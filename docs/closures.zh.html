The `wasm-bindgen` Guide

关闭

该#[wasm_bindgen]attribute支持将一些Rust闭包传递给JS. 你可以做的例子是:


# #![allow(unused_variables)]
#fn main() {
#[wasm_bindgen]
extern {
    fn foo(a: &Fn()); // could also be `&mut FnMut()`
}
#}

这是一个功能foo从JS导入,其中第一个参数是a堆栈关闭. 您可以使用a调用此函数&Fn()参数和JS将获得JS函数. 当. . . 的时候foo但是,函数返回时,JS函数将失效,并且将来使用它将引发异常.

闭包还支持参数和返回值,如导出,例如:


# #![allow(unused_variables)]
#fn main() {
#[wasm_bindgen]
extern {
    type Foo;

    fn bar(a: &Fn(u32, String) -> Foo);
}
#}

有时,不希望这些闭包的堆栈行为. 例如,您希望在JS中通过事件循环的下一轮安排闭包运行setTimeout. 为此,您希望导入的函数返回,但JS闭包仍然需要有效!

要支持此用例,您可以执行以下操作:


# #![allow(unused_variables)]
#fn main() {
use wasm_bindgen::prelude::*;

#[wasm_bindgen]
extern {
    fn baz(a: &Closure<Fn()>);
}
#}

该Closure类型定义在wasm_bindgen箱子并代表"长寿"封闭. JS封闭传递给了baz之后仍然有效baz返回,JS闭包的有效性与生命周期有关Closure在Rust. 一旦Closure被删除它将释放其内部内存并使相应的JS函数无效.

像堆栈封闭一样Closure也支持FnMut:


# #![allow(unused_variables)]
#fn main() {
use wasm_bindgen::prelude::*;

#[wasm_bindgen]
extern {
    fn another(a: &Closure<FnMut() -> u32>);
}
#}

这时你不能将一个JS闭包传递给Rust,在有限的情况下,你只能将一个Rust闭包传递给JS.
